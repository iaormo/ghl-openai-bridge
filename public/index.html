Bella's Playground — Documentation

Overview

Bella's Playground lets you test and configure your OpenAI-powered GHL assistant in real time. You can chat with the bot, tweak model parameters, manage function tools, add calendars, define custom fields, and edit the system prompt — all from one interface.

Getting Started

  • Go to Settings tab and enter your OpenAI API Key (starts with sk-)
  • Enter your Assistant ID (starts with asst_) — this is found in your OpenAI dashboard under Assistants
  • Click Save on each — the system prompt will auto-load from your assistant
  • Start chatting in the center panel to test your bot

Model Settings (Left Sidebar)

  • Model — Select the OpenAI model (gpt-4o-mini, gpt-4o, gpt-4-turbo, gpt-3.5-turbo)
  • Temperature (0–2) — Controls randomness. Lower = more deterministic, higher = more creative. Default: 1.0
  • Top P (0–1) — Nucleus sampling. Lower values restrict to more likely tokens. Default: 1.0
  • Max Tokens (100–4000) — Maximum length of the assistant's response
  • Contact ID — The GHL contact ID used for tool calls (e.g. booking, contact updates)
Tip: For consistent customer service responses, use Temperature 0.3–0.7. For creative/varied responses, use 0.8–1.2.

System Prompt (Prompt Tab)

The system prompt defines your assistant's personality, rules, and behavior. It is loaded from your OpenAI Assistant's instructions on startup.

  • Edit the prompt directly in the text area
  • Save Prompt — Persists changes back to the OpenAI Assistant (permanent)
  • Reset to Default — Reverts to the last saved version
  • If you edit without saving, changes only apply locally to the current session
Tip: Reference tool names in your prompt to guide when the AI calls them. Example: "When a customer asks about prices, call getProductCatalog before responding."

How Tools Work

Function tools let the AI perform actions — check calendars, update contacts, look up products, etc. When the AI decides it needs information or needs to take an action, it "calls" a tool and uses the result in its response.

The AI decides which tool to call based on:

  • The tool name — should be descriptive (e.g. getAvailableSlots)
  • The tool description — tells the AI when and why to use it
  • The parameters — what data the tool needs (defined as JSON Schema)
  • Your system prompt — you can instruct the AI to call specific tools in specific situations

Built-in Tools

These are pre-configured and ready to use:

  • getCurrentDate — Returns current Manila date/time
  • getContactInformation — Fetches customer info from GHL
  • updateContactInfo — Updates customer name, phone, or email in GHL
  • updateCustomField — Sets custom field values on the contact (orders, preferences, etc.)
  • getProductCatalog — Returns full product list with pricing and promos
  • getAvailableSlots — Checks open appointment times for a date
  • appointmentBooking — Books an appointment for the customer
  • getContactAppointments — Lists upcoming appointments
  • rescheduleAppointment — Moves an appointment to a new time
  • cancelAppointment — Cancels an existing appointment

Adding Custom Tools

Click + Add Tool in the Tools tab to define a new function tool.

  • Tool Name — camelCase, no spaces (e.g. checkOrderStatus)
  • Description — Be specific about when the AI should call this. The more context, the better.
  • Parameters (JSON Schema) — Define what data the tool accepts

Example — a tool to look up order status:

{
  "type": "object",
  "properties": {
    "order_id": {
      "type": "string",
      "description": "The order ID to look up"
    }
  },
  "required": ["order_id"]
}
Tip: Custom tools added here are for testing the AI's tool-calling behavior. The AI will attempt to call them, but unless you implement a backend handler, the result will be an "Unknown function" error. Use this to prototype tool schemas before implementing.

Enabling/Disabling Tools

In the left sidebar under Enabled Tools, you can toggle individual tools on/off with checkboxes. Disabled tools won't be sent to the AI, so it won't try to call them. This is useful for:

  • Testing how the AI behaves without certain tools
  • Isolating specific tool interactions
  • Reducing token usage by sending fewer tool definitions

Calendars

The Calendars tab shows the default GHL calendar and lets you add more. Each calendar needs:

  • Calendar ID — From your GHL location settings (Settings > Calendars)
  • Calendar Name — A friendly label for reference

Custom Fields

The Custom Fields tab manages which fields the updateCustomField tool can write to. Built-in fields (order details, payment, etc.) are pre-configured. Adding a new field here:

  • Updates the updateCustomField tool description so the AI knows about it
  • The field key must also exist in your GHL location (Settings > Custom Fields)

Example: Add key preferred_language with description "Customer's preferred language" — the AI can then save language preferences automatically.

Referencing Tools in Your Prompt

To get the best results from tool calling, explicitly guide the AI in your system prompt:

RULES:
- When a customer asks about products or prices,
  ALWAYS call getProductCatalog first.
- When the customer provides their name or phone,
  IMMEDIATELY call updateContactInfo.
- When the customer confirms an order, call
  updateCustomField for each order detail
  (product_interest, order_quantity, order_total,
  payment_method, shipping_address).
- Before suggesting appointment times, call
  getAvailableSlots for the requested date.
- Never make up prices — always use getProductCatalog.

Chat & Testing

  • Type a message and press Enter or click Send
  • Tool calls appear as expandable traces showing the function name, arguments, and result
  • Token usage (prompt + completion) is shown below the input
  • Clear resets the chat display (conversation history on the server is preserved per contact ID)
Tip: Change the Contact ID to test with different customer profiles. Use a real GHL contact ID to test with actual customer data.
